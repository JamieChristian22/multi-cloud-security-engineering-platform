# Runbook: High CPU Alert Response

## Trigger

CPU utilization exceeds the configured threshold.

## Response Steps

1. Confirm alert time and affected resource.
2. Review recent deployments or workload changes.
3. Check logs for application errors.
4. Validate whether the load is expected.
5. Scale, restart, or escalate based on impact.

## Closure Criteria

CPU returns below threshold and no active customer impact remains.
