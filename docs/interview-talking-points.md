# Interview Talking Points

## Why this portfolio matters

This portfolio shows practical cloud security engineering work, not just theory. Each project connects technical controls to business risk: access misuse, data exposure, misconfiguration, lack of audit evidence, and delayed incident response.

## How to explain the design

The architecture is split into four enterprise security domains: identity, monitoring, zero trust, and governance. This mirrors real security organizations where IAM, SOC, architecture, and GRC teams work together.

## Strong answer example

"I built this portfolio to show how I would secure a multi-cloud environment. I started with identity because identity is the control plane for cloud. Then I added monitoring and threat detection so changes and suspicious activity are visible. After that, I built zero trust and data protection controls to reduce exposure. Finally, I added governance and compliance so the environment can be audited and improved continuously."
